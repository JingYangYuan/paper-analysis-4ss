# 质性方法参考

本文档供 `modules/analysis/` 在执行质性编码、主题分析、内容分析、混合方法整合时按需读取。每次使用前先通过 §1 方法路由确定方法，再加载对应章节。

---

## 1. 方法路由

| 需求 | 方法 | 典型产出 | 理论传统 |
|------|------|---------|---------|
| 从材料中归纳新概念/理论 | 扎根理论 (Grounded Theory) | 核心范畴、理论命题 | 符号互动论、实用主义 |
| 已有框架，检验或套用到材料 | 框架分析 (Framework Analysis) | 矩阵、摘要表 | 政策研究、应用研究 |
| 提炼共享意义模式 | 主题分析 (Thematic Analysis) | 主题地图、主题叙述 | 现象学、解释主义 |
| 量化文本中框架/话题/态度频率 | 内容分析 (Content Analysis) | 编码频次表、交叉表、信度系数 | 传播学、社会心理学 |
| 解释因果机制（过程追踪） | 过程追踪 (Process Tracing) | 因果机制图、证据表 | 比较政治学、案例研究 |
| 多编码者一致性和可复现性 | 信度评估 | Cohen's κ / Krippendorff's α | 内容分析、系统编码 |
| 大规模文本初筛 | LLM 辅助编码 | 初编码、主题簇 | 计算方法、NLP |
| 解释回归机制或统计发现 | 混合方法整合 | joint display、案例选择矩阵 | 实用主义、批判实在论 |
| 分析叙事/生命史 | 叙事分析 (Narrative Analysis) | 叙事结构、身份建构 | 叙事心理学、社会学 |
| 分析话语/权力/知识关系 | 话语分析 (Discourse Analysis) | 话语策略、主体位置 | 后结构主义、批判理论 |

## 2. 编码单位与抽样

### 2.1 编码单位选择

| 单位 | 适用场景 | 示例 |
|------|---------|------|
| 词/短语 | 内容分析、关键词统计 | 政策文本中的"可持续发展"词频 |
| 句子 | 短开放题、政策条款 | 问卷中"为什么支持该政策" |
| 段落 | 访谈逐字稿、田野笔记 | 受访者讲述工作经历的一段话 |
| 话轮 | 对话、焦点小组 | 焦点小组中某成员的一轮发言 |
| 事件/情节 | 案例研究、过程追踪 | 某政策从提案到落地的完整过程 |
| 文档 | 组织文件、新闻报道 | 某企业的年度可持续发展报告 |

### 2.2 抽样策略

| 策略 | 说明 | 适用 |
|------|------|------|
| 目的性抽样 (Purposive) | 按理论相关性选择个案 | 典型案例、关键案例 |
| 最大变异抽样 | 覆盖最广泛的变异范围 | 异质性大的总体 |
| 极端/偏差案例 | 选择极端值或异常值 | 检验理论边界 |
| 滚雪球抽样 | 通过受访者推荐下一个 | 隐藏或难以接触的人群 |
| 理论抽样 | 边收集边分析，按新兴理论引导下一步 | 扎根理论 |
| 分层目的抽样 | 在子群体内做目的性抽样 | 跨群体比较 |

### 2.3 饱和度判断

- **理论饱和 (Theoretical Saturation)**：新数据不再产生新的理论见解或范畴属性。扎根理论的核心停止标准。
- **主题饱和 (Thematic Saturation)**：新数据不再产生新主题。适用于主题分析和框架分析。
- **数据饱和 (Data Saturation)**：没有新的信息/模式出现。最保守的标准。
- 实际操作中，建议在预计数据收集量的 80% 时做一次饱和评估，决定是否需要补充。

## 3. 扎根理论 (Grounded Theory)

### 3.1 三大学派

| 学派 | 代表学者 | 编码步骤 | 核心特征 |
|------|---------|---------|---------|
| 经典扎根 | Glaser & Strauss (1967) | 开放→选择→理论编码 | 不预设框架，让理论从数据中涌现 |
| 程序化扎根 | Strauss & Corbin (1990/1998) | 开放→主轴→选择编码 | 引入编码范式模型（条件-行动-结果） |
| 建构扎根 | Charmaz (2006/2014) | 初始→聚焦→理论编码 | 强调研究者与数据的共建，接受多元解释 |

### 3.2 编码范式模型 (Strauss & Corbin)

主轴编码围绕"编码范式"展开六个维度，将碎片化的开放编码整合为连贯结构：

```
因果条件 → 核心现象 → 脉络条件 → 中介条件 → 行动/互动策略 → 结果
```

编码范式在主轴编码阶段用于发现并阐明各编码范畴之间的相互联系，帮助研究者将分散的开放编码系统地串联成一个逻辑整体。

### 3.3 编码操作流程

1. **开放编码**：逐行阅读，用动名词短语标注（如"争取资源""回避问责"），保持贴近数据
2. **主轴编码**：用编码范式模型将开放编码组织成"范畴-子范畴"层级结构
3. **选择编码**：识别能统领多数范畴的核心范畴，撰写故事线将各范畴串联
4. **理论饱和**：继续收集数据直到新数据不再产生新的范畴属性

### 3.4 持续比较法 (Constant Comparison)

扎根理论的核心分析引擎，贯穿所有编码阶段：

- 事件与事件比较 → 生成概念
- 概念与事件比较 → 精炼概念属性
- 概念与概念比较 → 形成范畴
- 范畴与范畴比较 → 确立关系和理论边界

### 3.5 备忘录 (Memo-writing)

- 记录分析决策：为什么生成这个编码、为什么合并这些范畴
- 记录理论直觉和暂时性假设
- 记录与其他文献的对话（延迟文献回顾并非不看文献）
- 最终备忘录是论文初稿的核心原材料

## 4. 主题分析 (Thematic Analysis)

### 4.1 Braun & Clarke 六步法 (2006, 2021)

| 步骤 | 操作 | 产出 |
|------|------|------|
| 1. 熟悉数据 | 反复阅读、转写、做初步笔记 | 数据熟悉笔记 |
| 2. 生成初始编码 | 系统编码所有相关数据片段 | 初始编码列表（100-300 个） |
| 3. 寻找主题 | 将编码归类为候选主题 | 候选主题 + 子主题地图 |
| 4. 审查主题 | 检查主题内部连贯、主题之间区分清晰 | 主题地图 v2，合并/拆分/丢弃 |
| 5. 界定与命名 | 为每个主题写定义和边界 | 最终主题名称 + 定义段落 |
| 6. 撰写报告 | 用生动引文和分析叙述每个主题 | 论文主题分析章节 |

### 4.2 三种取向

| 取向 | 编码方式 | 理论关系 |
|------|---------|---------|
| 归纳 (Inductive) | 由数据驱动，不预设编码框架 | 与扎根理论接近但更灵活 |
| 演绎 (Deductive) | 由理论/研究问题驱动，预设编码模板 | 与框架分析接近 |
| 批判/潜伏 (Critical/Latent) | 分析话语背后的意识形态和权力结构 | 与话语分析接近 |

### 4.3 主题分析质量自检 (Braun & Clarke, 2006)

- 编码不是简单的数据摘要，而是分析性标签
- 主题不是"话题"(topic)，而是围绕某个核心意义/模式的解释
- 每个主题的内部一致性和主题间区分清晰
- 引文占分析文本比例不超过 30%，分析占 70%
- 是否有反例或例外案例被讨论？

## 5. 框架分析 (Framework Analysis)

### 5.1 适用场景

- 应用政策研究，需要在短期内生成可操作建议
- 多学科团队合作，需要透明系统的方法
- 有初步研究问题但数据收集前可以预设关键议题
- 多案例比较分析

### 5.2 五步法 (Ritchie & Spencer, 1994)

| 步骤 | 操作 |
|------|------|
| 1. 熟悉 | 通读数据，标记关键议题 |
| 2. 建立框架 | 根据研究问题+数据反复构建编码框架 |
| 3. 标引 | 将框架应用到全部数据（可使用 NVivo/MAXQDA/Atlas.ti） |
| 4. 制图 | 创建按主题×案例的矩阵，每个单元格放摘要 |
| 5. 映射与解释 | 在矩阵中寻找模式、关联和解释 |

### 5.3 框架矩阵示例

| 案例 | 主题1：政策理解 | 主题2：执行障碍 | 主题3：适应策略 |
|------|---------------|---------------|---------------|
| A校 | "认为减负就是少布置作业" | 家长投诉、教师惯性 | 增加课堂效率 |
| B校 | 将减负理解为课程整合 | 评价体系未同步 | 跨学科项目 |

## 6. 内容分析 (Content Analysis)

### 6.1 定量 vs 定性内容分析

| 维度 | 定量内容分析 | 定性内容分析 |
|------|-------------|-------------|
| 目标 | 系统量化文本特征 | 解释文本中的意义模式 |
| 编码 | 预定义编码本，独立编码 | 编码本可迭代演进 |
| 信度 | 必须报告 κ 或 α | 可选，不强制 |
| 分析 | 频次、交叉表、卡方检验 | 主题归纳、模式识别 |
| 代表 | Berelson (1952), Krippendorff (2018) | Schreier (2012), Hsieh & Shannon (2005) |

### 6.2 定性内容分析的三种路径 (Hsieh & Shannon, 2005)

| 路径 | 起点 | 编码来源 |
|------|------|---------|
| 常规 (Conventional) | 数据驱动，没有预设理论 | 从数据中归纳 |
| 定向 (Directed) | 已有理论/框架指导 | 理论演绎 + 数据补充 |
| 总结 (Summative) | 关键词识别→语境解释 | 先量化后质性 |

### 6.3 抽样要求

- 内容分析的核心抽样原则：**分析单位**必须明确（词/句/段/篇/人物/事件）
- 时间范围必须指定（如 2015-2025 年政府工作报告）
- 来源范围必须指定（如《人民日报》+ 31 省党报）
- 样本量论证：数据来源何时达到信息饱和

## 7. 编码本设计规范

### 7.1 编码本要素

每条编码必须包含：

1. **编码名称**：简短标签（最大化 4-5 词）
2. **清晰定义**：该编码捕捉什么现象
3. **纳入标准**：什么文本属于该编码（必要条件）
4. **排除标准**：什么文本不属于该编码（边界）
5. **典型例子**：至少 2 条，附原文摘录
6. **边界案例**：模棱两可的情况及处理规则
7. **与相近编码的区别**：如"焦虑"≠"恐惧"的区别
8. **修订记录**：日期 + 修订理由

### 7.2 编码本演进

- V1.0：基于前 20% 数据归纳生成，约 20-50 个编码
- V2.0：应用到 50% 数据后，合并/拆分/新增
- V3.0：终版编码本（人工复核后可锁定的版本）
- 编码本每一次演化需记录：哪些编码被合并、拆分、新增、删除，以及理由

## 8. 信度评估

### 8.1 常用指标

| 指标 | 适用 | 阈值 |
|------|------|------|
| Cohen's κ | 两个编码者，非加权 | < 0.60 不足 / 0.60-0.69 勉强 / 0.70-0.79 可接受 / ≥ 0.80 高 |
| Fleiss' κ | 三个及以上编码者 | 同上 |
| Krippendorff's α | 任意数量编码者，支持多种数据类型 | ≥ 0.667 可接受 / ≥ 0.80 良好 |
| 百分比一致 (PA) | 简单快速但不纠正机遇 | ≥ 80% 辅助参考 |

### 8.2 信度操作流程

1. 培训编码者，用编码本 v1.0 练习 10-20% 的数据
2. 计算信度，识别分歧来源
3. 讨论分歧，修订编码本（记录修改理由）
4. 对独立验证样本重新计算信度
5. 信度达标后，剩余数据可由单人编码（如资源有限）

### 8.3 信度说明

- 主题分析和扎根理论**可以**不强制报告 κ/α，但必须在方法部分说明反身性和分析决策过程
- 内容分析和系统编码**必须**报告信度指标
- LLM 辅助编码必须由人工核验一致性
- 反身性声明不是缺陷，而是质性研究的质量标志

## 9. LLM 辅助编码协议

### 9.1 适用场景

- 大规模文本（> 100 篇文档），人工编码不可行
- 初始编码生成（加速扎根理论的开放编码阶段）
- 探索性分析（确认值得深入人工分析的方向）

### 9.2 操作流程

1. **去标识化**：所有文本先人工去标识化（姓名/机构/地点/联系方式），不可将含个人身份的文本直接输入外部 LLM API
2. **编码本注入**：将编码本完整写入 prompt，包括定义、纳入/排除标准、典型例子
3. **LLM 初编码**：逐段请求编码，要求返回"编码 + 原文摘录 + 置信度"
4. **人工核验**：随机抽取 20-30% 的 LLM 编码结果，由人工独立编码做交叉验证
5. **分歧处理**：LLM 与人工不一致的区段，人工重编码并修订 prompt
6. **最终报告**：报告 LLM 编码-人工编码一致率，标注 LLM 辅助的局限性段落

### 9.3 风险与限制

- LLM 可能生成不存在的编码（幻觉）→ 必须有人工核对
- LLM 可能受训练数据偏见 → 敏感话题（种族、性别、宗教）应优先人工编码
- LLM 缺乏文化语境 → 特定文化/方言文本应人工编码
- 编码者间的信度评估不能替代 LLM-人工的一致性报告

## 10. 过程追踪 (Process Tracing)

### 10.1 目标

系统检验因果机制的每个环节是否在案例中实际发生。不是描述故事，而是用证据检验机制。

### 10.2 四类证据 (Beach & Pedersen, 2013)

| 证据类型 | 含量 | 检验力 |
|---------|------|--------|
| 风吹草动 (Straw-in-the-Wind) | 通过不必然，不通过不排除 | 弱 |
| 圈套证据 (Hoop) | 通过不必然，不通过排除 | 中 |
| 枪口证据 (Smoking Gun) | 通过则确认，不通过不排除 | 中 |
| 双决证据 (Doubly Decisive) | 通过确认，不通过排除 | 强 |

### 10.3 操作步骤

1. 理论化：将因果机制分解为 n 个离散步骤（事件 A→B→C→D）
2. 对预测每个步骤的证据做"四类证据"分类
3. 收集每个步骤的经验证据
4. 对每个步骤做贝叶斯更新：先验置信度 × 证据检验力 → 后验置信度
5. 报告机制各步骤的证据支撑表

## 11. 混合方法整合

### 11.1 Joint Display

定量发现与定性主题的并排对照表，是混合方法论文的核心交付物：

| 定量发现 | 定性主题 | 整合解释 | 元推论 |
|---------|---------|---------|--------|
| 回归系数 β=0.34*** | 受访者描述"政策让我敢花钱了" | 统计效应与生活经验一致 | 收敛 |
| 交互项不显著 | 访谈发现性别差异 | 统计检验力不足或真实无差异 | 补充 |
| PSM 后 ATT 不显著 | 某些受访者描述明显变化 | 平均效应掩盖异质性 | 解释 |

### 11.2 整合策略

| 策略 | 说明 | 适用 |
|------|------|------|
| 收敛 (Convergence) | 定量与定性发现方向一致 | 增强研究发现的可信度 |
| 补充 (Complementarity) | 定量与定性覆盖同一问题的不同方面 | 充实理解 |
| 解释 (Explanation) | 定性用于解释定量发现的机制 | 量化→质性 |
| 发展 (Development) | 前一阶段发现指导后一阶段的数据收集 | 顺序设计 |
| 扩展 (Expansion) | 定量和定性分别回答不同但有联系的问题 | 同时性设计 |

### 11.3 案例选择 (嵌套设计)

定量分析后，基于量化结果选择质性子样本：

- 典型案例 (Typical)：选择接近回归线的案例
- 异常案例 (Deviant)：选择偏离回归线的异常值
- 关键案例 (Critical)：选择最能检验理论的案例
- 最大变异 (Maximum Variation)：选择分布在分布两端的案例

## 12. 引文使用规则

| 规则 | 说明 |
|------|------|
| 数量 | 每个主题保留 2-5 条代表性匿名引文 |
| 长度 | 短引文 1-3 句融入正文，长引文（>40 词）缩进单独成段 |
| 分析先行 | 引文前应有分析导向句，引文后必须有分析句 |
| 不能替代 | 引文是证据，不替代作者的解释和论证 |
| 去识别化 | 不展示姓名、单位、具体地点、独特事件细节 |
| 少数案例 | 与主题不符的负面案例必须单独说明，不能沉默 |
| 编辑标记 | 省略用 `[...]`，插入解释用 `[xxx]` |

## 14. 报告清单

质性分析最终交付必须包含：

1. **方法选择与论证**：为什么选择该方法，为什么其他方法不合适
