# 质性方法速查

## 1. 方法路由

| 需求 | 方法 |
|---|---|
| 从访谈中生成概念 | 扎根理论开放编码 |
| 已有理论框架，要检验材料 | 演绎编码或框架分析 |
| 提炼共享意义模式 | 主题分析 |
| 大批文本中统计框架/话题/态度 | 内容分析 |
| 多编码者一致性 | 信度评估 |
| 大规模文本初筛 | LLM 辅助编码 + 人工验证 |
| 解释回归机制 | 混合方法 joint display |

## 2. 编码单位

- 句子：适合短开放题和政策文本。
- 段落：适合访谈和田野笔记。
- 话轮：适合对话和焦点小组。
- 事件：适合案例研究和过程追踪。
- 文档：适合组织文件和新闻样本。

## 3. 编码本质量

好编码本必须有：

- 清晰定义。
- 纳入标准。
- 排除标准。
- 典型例子。
- 边界案例。
- 与相近编码的区别。
- 修订日期和修订理由。

## 4. 信度阈值

| 指标 | 解释 |
|---|---|
| < 0.60 | 不足，需重训或修订编码本 |
| 0.60-0.69 | 勉强可用，需说明限制 |
| 0.70-0.79 | 常规可接受 |
| >= 0.80 | 较高一致性 |

反身性主题分析不必强制追求高 kappa，但若声称系统内容分析或可重复编码，必须报告信度。

## 5. 引文使用

- 每个主题保留 2-5 条代表性匿名引文。
- 引文后必须有分析句，不能让引文替代解释。
- 不展示可识别身份、地点、单位或罕见事件细节。
- 少数/负面案例要单独说明。
