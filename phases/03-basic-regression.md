# Phase 03: 基础回归、稳健性与结果表

## 1. 模型路由

先读取 `modules/analysis/references/model-router.md`。根据因变量类型、数据结构和研究问题选择基础模型。

## 2. 主模型规范

每个主模型必须记录：

- 样本口径和 N。
- 因变量、核心自变量、控制变量。
- 固定效应和聚类层级。
- 权重使用与否。
- 标准误类型。
- 表格展示格式：系数下方必须放 t/z 统计值，不放标准误。
- 数值格式：系数、边际效应、t/z 统计值和比例统一保留 3 位小数；N 和频数保留整数。
- 模型族和链接函数。

## 3. 模型决策记录

运行主模型前，先生成或追加 `analysis-output/reports/model-decision-[date].md`：

```markdown
## 模型决策：[模型名]

- 研究问题：
- 因变量类型：
- 数据结构：
- 核心解释变量：
- 模型选择：
- 标准误/聚类层级：
- 固定效应：
- 权重：
- 可解释为因果吗：
- 必做稳健性：
- 不足与风险：
```

模型选择必须能追溯到 `modules/analysis/references/model-router.md`。

## 4. 常用模型

| 任务 | 模型 |
|---|---|
| 连续因变量 | OLS、稳健/聚类标准误、固定效应 |
| 二元因变量 | logit/probit + AME/预测概率 |
| 有序因变量 | ordered logit/probit + 边际效应 |
| 多分类因变量 | multinomial logit + 预测概率 |
| 计数因变量 | Poisson/NB，检查过度离散 |
| 面板数据 | FE/RE、双向固定效应、Hausman |
| 基础因果 | DiD、事件研究、PSM、IV/2SLS |

## 5. 诊断顺序

主模型后按顺序做诊断：

1. 样本量和变量缺失是否与清洗报告一致。
2. 系数方向是否因加入控制变量或固定效应剧烈翻转。
3. 标准误层级是否与数据结构一致。
4. 非线性模型是否输出 AME 或预测概率。
5. 交互项是否给出边际效应图或简单斜率。
6. 面板模型是否说明组内变化来源。
7. 因果设计是否完成必要识别诊断。

诊断不通过时，不直接进入写作；先返回清洗或模型选择。

## 6. 稳健性

至少考虑：

- 替代因变量或核心自变量口径。
- 替代样本限制。
- 替代标准误或聚类层级。
- 加入/移除固定效应。
- 分组回归或交互项。
- DiD 平行趋势、安慰剂、窗口敏感性。
- IV 第一阶段、弱工具变量、过度识别检验。

稳健性不是越多越好。优先选择能回应理论争议、测量争议和识别风险的检验；不要机械堆积显著性表。

## 7. 结果解释规则

- 不把相关性写成因果，除非研究设计支持因果识别。
- logit/probit 结果主要解释 AME 或预测概率。
- 交互项优先报告边际效应图或简单斜率。
- 固定效应模型说明估计来自组内变化。
- 表格星号只能辅助阅读；正文报告方向、量级和置信区间或 exact p；表格括号内为 t/z 统计值。
